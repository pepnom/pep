word by word parsing with nom

parsing word-by-word with the ℕ𝕠𝕞 language

nom includes several commands for lexing (or “scanning” ) the input-stream. These include the most fundamental which is read and also until , while and whilenot . The last 3 words are designed to create parse-tokens from chunks of text (or chunks of the input-stream).

The until command is convenient in cases where the end delimiter is more than one character. until also has the advantage that it automatically ignores an end delimiter if that delimiter is “escaped” with the escape character (which is \\ by default).

parsing words

But for parsing words the while and whilenot commands are more useful.

notes

ℕ𝕠𝕞 does not currently allow conjunction sets of characters which means that you either have to explicitly name every character that you wish to include in your word* token, or else use one of the predefined character classes such as [:alpha:] or [:alnum:]. This is a significant limitation which is seen in the nom XML parser/error-checker because “names” in xml syntax can contain '.' '-' or '_' or any UNICODE alphabetic character. But there is no easy way to add these three characters to the [:alpha:] unicode class.
