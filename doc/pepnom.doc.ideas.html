unicode and reading

Unicode doesn't actually define what is a 'character', instead it uses code points and combining marks (diacritics, underlines, etc). So the question of what the read command should do is a bit tricky. Should it read a byte, or a (utf8) code point or a grapheme cluster (the series of codepoints and combining marks that makes up a single visible character) ? Usually the answer is the last: A grapheme cluster, but not always.

So the read command could have several modes, and it could even read the data as a byte and treat it as 'binary'.
